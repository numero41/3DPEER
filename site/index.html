Guide

What it does

3DPEER turns a 3D model into a single .html file that carries the model, its decompressor and its viewer. The file opens with a double-click on any device with a browser, from a mail attachment, a USB stick or a chat message. It needs no app, no plugin, no account and no internet, and it will still open years from now.

Compress up to 100×

The built-in pipeline compresses geometry and textures on your machine. Heavy production files routinely come out 10 to 100× smaller. The defaults are visually safe, the estimated output updates live, and the compare view shows original and compressed side by side in the viewport:

  • geometry precision: how finely the shape is kept. Lower is smaller, and very low can look faceted.
  • shading precision: how finely surface shading is kept.
  • texture size and quality: cap texture dimensions and recompression quality. Quality 100 keeps your originals.
  • decimate: removes a share of triangles. Off by default, so your geometry is never touched unless you ask.

Annotate and collaborate

Pin numbered notes directly on the model before exporting. The recipient sees them, can answer with their own pins, and saves an annotated copy that travels back to you like any file. A full review round happens with no platform in between.

Present your work

Files can ship camera views, material presets, lighting controls and animation playback, so a portfolio piece or a client delivery is explorable, not just visible. Script-blocked previews (webmail, phone mail apps) still show a built-in four-view snapshot of the model.

Asset validation for teams

Because one file holds the whole review, validation stays simple: send the compressed artifact, collect annotated copies, compare rounds. The same file embeds cleanly in an intranet or portfolio page for showcasing approved assets. Enterprise options such as batch tooling, shared note threads and self-hosted bundles are on the roadmap. Write to us if you need them.

Nothing is stored on this site

The site has no upload endpoint, so there is nowhere to store your model even by accident. Import, compression, annotation and export all run inside your browser, and the files you generate make no network requests. Close the tab and nothing remains.

Supported formats

Drop a glb, gltf, obj, stl, ply, fbx or usdz file. Non-glTF formats are brought in automatically. Fidelity on materials and textures can vary with how the original file was produced.

Sharing the file

The exported file is self-contained. Send it like any attachment, over mail, WhatsApp, Drive or Slack. Two mail-client behaviours are worth knowing, both by design of the mail providers:

  • Gmail on desktop shows HTML attachments as source text in its preview. Recipients should download the file and open it, and it launches in their browser.
  • iPhone Mail previews attachments with scripts disabled: the recipient sees the built-in snapshot views and a short notice instead of the interactive model. Opening the file in a real browser restores full interaction.

Viewer controls in the file

When "ship viewer controls" is checked, the exported file carries the on-screen controls: camera views, material and lighting options, and any notes you added. Uncheck it for a minimal orbit-only file. Animation playback ships whenever the model has clips.

Licences and the watermark

Every feature is free: all formats, all compression settings, no size limit, no account required. Files you generate carry a small visible watermark and a "made with 3DPEER" footer, and buying a licence is the one thing that removes them. A licence is a signed key you paste into the app. It is checked on your machine, offline. No account lookup, no phone-home, and nothing about your models or your licence leaves the browser.

The watermark is a deterrent, not a lock. An exported file is an ordinary HTML document that has to run in any browser, ten years from now, with no server to check anything, so anyone determined enough can edit it. We would rather say that plainly than pretend otherwise. In practice the watermark makes casual reuse obviously unlicensed, and licence terms cover the rest: automated or bulk processing, and redistribution of unwatermarked files, require a commercial licence.
