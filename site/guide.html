3DPEER

Guide

What it does

3DPEER turns a 3D model into a single .html file. The model, its decompressor and its viewer travel together: the file opens with a double-click, works offline, and will still work years from now. Nothing is ever uploaded — every step runs in your browser.

Supported formats

glb and gltf are native. obj, stl, ply, fbx and usdz are converted on the fly. FBX materials are approximated on the web. USDZ reads ASCII layers and binary usdc crates, including packages nested inside packages; material and texture fidelity on complex production files is still approximate. If a file yields no geometry, you will get a clear warning instead of an empty viewport.

Compression

The export pipeline compresses geometry, animations and textures locally. The defaults are visually safe. In export settings:

  • geometry precision — how exactly vertex positions are stored. Lower is smaller; very low values can facet large models.
  • shading precision — accuracy of surface directions; affects highlight smoothness.
  • texture size / quality — textures are scaled down to the given edge and re-encoded as WebP. Quality 100 keeps your originals.
  • decimate — removes a share of triangles. Off by default: your geometry is never touched unless you ask.
  • auto — give a target size in MB and the best quality that fits is found for you.

Sharing the file

The exported file is self-contained — send it like any attachment (mail, WhatsApp, Drive, Slack). Two mail-client behaviours to know about, both by design of the mail providers:

  • Gmail on desktop shows HTML attachments as source text in its preview. Recipients should download the file, then open it — it launches in their browser.
  • iPhone Mail previews attachments with scripts disabled: the recipient sees the built-in snapshot views and a short notice instead of the interactive model. Opening the file in a real browser restores full interaction.

Each export embeds a four-view snapshot grid, so even script-blocked previews show your model.

Viewer controls in the file

When "ship viewer controls" is checked, the exported file carries camera view presets, a material override (original, clay, metallic, glass, matcap, wireframe) and lighting sliders. Uncheck it for a minimal orbit-only file.

Privacy

The site has no upload endpoint. Models never leave your device; the exported file makes zero network requests, ever.
